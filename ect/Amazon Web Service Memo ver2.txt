Cloud 6가지 이점
- 초기 선투자 비용 없음
- 운영 비용 절감
- 탄력적인 운영 및 확장
- 속도 및 유연성
- 핵심 업무에만 집중 가능
- 글로벌 확장

리전 : AWS 서비스가 운영되는 지역으로 복수개의 데이터 센터들의 집합
가용 영역 : 리전 내 위치한 복수 개의 데이터 센터들로 각각 물리적으로 분리되어 있어
고가용성 / 이중화 구성의 기본 요소


멀티 클라우드

서로 다른 업체에서 2개 이상의 퍼블릭 클라우드를 이용해 하나의 서비스를 운영하는 것
- 업체 종속성 탈피
	사용하고 있는 클라우드가 망하거나 가격 인상등에 대비
- 단일 클라우드의 취약점 보안
- 워크로드에 따른 맞춤 선택
	작업방향에 따라 적절한 클라우드를 사용
( 퍼블릭 클라우드와 프라이빗 클라우드를 같이 사용하는 경우가 많다. )

모놀리식 아키텍처는 단 하나의 서버를 이용해서 서비스를 한다면
마이크로 서비스 아키텍처는 여러 개의 서버를 이용해서 각각의 서비스를 진행하게 하여
빠른 처리를 한다.
[ 예를 들어 세 가지 서비스를 운영하고 있다면 모놀리식 아키텍처는
	하나의 서비스를 변경하고 싶을 때 변경이 힘들며
	세 서비스 중 하나의 서비스에만 과부화가 걸릴 때 하나만
	처리할 수가 없다. 하지만 마이크로 서비스 아키텍처는 가능하다. ]
