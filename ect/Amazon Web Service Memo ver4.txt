스토리지 서비스 - Amazon S3 및 Amazon EBS

S3 ( Amazon Simple Storage Service )
- 인터넷용 스토리지
- 기본적으로 온라인, HTTP 기반 액세스 - 접근 방식이 http
- 웹을 통해 언제 어디서든 원하는 양의 데이터를 저장하고 검색할 수 있는 스토리지
- 고도의 확장성, 안정성, 속도 및 내구성

업로드 방법 - 파일을 다운로드하고 고치고 다시 업로드 하는 것과 같은 방식으로 업로드 한다.
버저닝 - 실수로 삭제해서 데이터가 날아갈 것을 방지한다.

따라서 S3는 데이터를 여러 번 수정해야할 때는 불리하다.

~~~~~~~~~~~~~~~~~~~~~~~~~~~

EBS
외장하드와 비슷한 형식으로 사용
데이터를 여러 번 수정해야할 떄 사용한다.

SSD 지원 볼륨 :
	I/O 크기가 자곡 읽기/쓰기 작업 빈도가 높은 트랜잭션 워크로드에 적합
	IOPS 성능이 뛰어남 - IO가 빨리 된다.
HDD 지원 볼륨 :
	대규모 스트리밍 워크로드에 적합
	처리량이 뛰어남 - 큰 데이터를 한 번에 읽어들이고 한 번에 저장
따라서 SSD가 무조건 좋은 것은 아니다.

~~~~~~~~~~~~~~~~~~~~~~~~~~~

EC2 인스턴스 스토리지
- 무료 로컬 직접 연결 블록 스토리지
- IOPS에 최대로 적합함
- 하지만 데이터가 휘발성임 ( 날라감 )
- 엄청난 트래픽이 필요하고 결과만 뽑아내면 되는 경우에 사용
